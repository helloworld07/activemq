1.这是一个简单jms消息队列的应用，需继续深挖其中的内容：
 1.可以切换队列模式和订阅者模式。
    两种模式的发布方式基本相同，Destination需要有所变化；
    主要在applicationContext-activemq.xml中修改（包括监听器和模板）
 2.主要原理：
    消息生产者将消息发送给第三方，这里用的ActiveMQ，进行暂时存放（localhost:61616）
    消息消费者将消息从第三方取出。
    在发送时不会等待，也不受消息接收方的处理效率影响（------有没有实例，最好模拟一个---------）
 3.这里用到了jmsTemplate模板，可以方便的进行发送和接收处理。
